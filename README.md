# Summer Movie Wager 2019
## Serverless scrapping and Gatsby deployment via AWS Amplify

### Description
The intention of this repo is to:
  * Use a serverless project to scrap data from a specified site
  * Use the scrapped data to generate a static site using Gatsby.js

This was due to me wanting a different way of visualizing the data that was available from https://thesummermoviewager.com in a easier to comprehend manner for the people who I was playing with. This was my first project also into using Gatsby and trying to understand how it works.

For more information about The Summer Movie Wager, visit [here](https://thesummermoviewager.com/help.php)

#### The Serverless Functionality
The serverless portion of this project has its own directory located [here](scraper-serverless/readme.md).

What this scrapper does is the following:
  * Sets up a cron schedule for a lambda function to run
  * The lambda function uses a combination of [cheerio](https://www.npmjs.com/package/cheerio) and [puppeteer](https://www.npmjs.com/package/puppeteer) to visit the specified Summer Movie Wager page (the page which contains you and your team's movie picks) and selects certain DOM elements, parses them, and saves them to a S3 Bucket.
  * :warning: The cron schedule can be any cron expression you want, but I only triggered it to run once a day to be kind to the original website. :warning:

```
Cron -> Lambda -> S3
```

#### Gatsby Framework
The Gatsby portion of this project requests the data to populate the site via the S3 bucket. The S3 Bucket contains the JSON object that is generated by the serverless component.

```
S3 -> Gatsby
```

Gatsby Configurations required:
  * `.env.production` and `.env.development` are used for these values:
    * `GATSBY_SUMMOVWAG_LOC` - The location of the S3 bucket that contains the data required to populate the site
    * `GATSBY_SUMMOVWAG_SITE` - The location of the original Summer Movie Wager site for your specific team.

Through AWS Amplify, you can specific [Environment Variables](https://docs.aws.amazon.com/amplify/latest/userguide/environment-variables.html) via their GUI. Any enviornment variables set locally must be replicated via the Amplify Console.

For posterity, I've chosen to keep the players and their picks in this repo via the [playersPicksList.js](src/components/playersPicksList.js) file.

Feel free to clone this repo and have this data fetched in a similar fashion with how Gatsby is fetching the data to populate the site.

Images used within this repo are not mine, but were taken from BoxOfficeMojo.com. These images represent a player's specific picks in an easier to understand fashion.
